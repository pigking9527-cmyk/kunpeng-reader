# 电子书阅读器（Tauri 版）开发文档

> 一个以 Windows 为主线的高性能本地电子书阅读器。书架与阅读页相互独立、EPUB 原生渲染、按章虚拟化加载，大书秒开。Android / macOS 后续另起工程，不再混入当前 Windows 项目。

---

## 一、技术栈

| 层 | 选型 | 说明 |
|----|------|------|
| 后端 / 框架 | **Rust + Tauri 2.x** | 业务逻辑、文件解析、窗口管理跑在 Rust 端 |
| 前端 / UI | **系统 WebView2 + 原生 HTML/CSS/JS** | 不引入前端框架，纯手写，体积小、启动快 |
| 进程模型 | Tauri 多窗口（`WebviewWindowBuilder`） | 书架窗口与每个阅读窗口彼此独立 |
| 数据传递 | 自定义 URI 协议 + `#[tauri::command]` | 按章、按资源异步拉取数据 |

主要依赖：`tauri`、`tauri-plugin-dialog`、`serde`、`epub`、`image`、`encoding_rs`、`chardetng`、`flate2`、`pulldown-cmark`、`mobi`、`rusqlite`、`fastembed`、`instant-distance`、`pdf-extract`、`lopdf`、`tokio-tungstenite`。

发布优化：v1.8.6 使用 `opt-level=3`、`lto="thin"`、`codegen-units=16`、`incremental=true`，保留运行优化，同时加快日常 release 构建。固定脚本 `scripts/check.ps1` 负责检查，GitHub Actions 在 `main` 的 push / pull request 上运行同一套检查；`scripts/build-release.ps1` 负责发布构建、复制单个桌面 exe、校验图标并刷新 Windows 图标缓存；GitHub Release 资产需要同时上传单文件绿色版和 NSIS 安装包。

## 三、核心架构

### 数据通路

前端从自定义 URI 协议异步取数据：

| 路由 | 作用 |
|------|------|
| `/cover/<id>` | 封面图片 |
| `/book/<id>` | 阅读页外壳 HTML |
| `/chapter/<id>/<i>` | 单章 JSON `{head, body}`，按需逐章拉取 |
| `/txt/<id>` | TXT / Markdown / MOBI / AZW / AZW3 正文 |
| `/res/<id>/<path>` | EPUB 内部资源 |
| `/pdf/<id>` | PDF 原文件 |

控制类操作（打开书、读信息、存进度、搜索、书签、改简介、同步等）走 `#[tauri::command]`。

### 关键定论

- 重命令必须 `async`，避免 UI 主线程卡死。
- 书 ID 是 `u64`，前端必须按字符串传递，避免超过 JS 安全整数后精度丢失。
- EPUB 大书性能依赖按章虚拟化：只渲染当前章，切章时再拉下一章。
- 书架启动阶段要少做磁盘扫描和逐本检查；封面、自动导入、字数统计都应延后或后台执行。
- Windows 图标显示可能被 Explorer 缓存影响，发布脚本必须校验 exe 内部图标并刷新图标缓存。
- 阅读页注入脚本不再直接塞进 Rust 大字符串，统一放在 `ui/reader-page-head.html`，由 `src/reader_page.rs` 通过 `include_str!` 引入。

## 四、功能清单

### 书架

- 批量导入 EPUB / PDF / TXT / Markdown / MOBI / AZW / AZW3。
- 封面墙、横向列表、评分、阅读进度、封面显示设置。
- 搜索框带历史记录和清空按钮；可切换普通书架过滤 / 全书架正文检索。
- 漏斗面板：排序、布局、阅读状态过滤、评分过滤。
- 账号窗口：注册、登录、退出登录、本机多账号保存、立即同步。
- 阅读统计：日 / 月 / 年 / 总视图、贡献图、统计项开关、时间 / 字数柱状图切换、图表与当日书籍列表。
- 笔记汇总：按书汇总高亮、批注、查词记录，并导出 Markdown。
- 数据包导入 / 导出：备份或迁移轻数据。

### 阅读页

- 目录、搜索、信息、朗读、批注/高亮、全屏、设置、生词本。
- EPUB CSS 多栏分页；PDF 使用 PDF.js 连续滚动与文字层。
- 书签、续读、跨章链接、脚注弹窗、主题和排版设置。
- 离线词典支持中中 / 中英 / 英中 / 英英查询。
- 划词翻译支持 DeepL / Google / 百度 / 腾讯翻译配置，凭据由用户本机填写保存。
- 生词本居中浮层，双列瀑布流，支持查询次数、排序、学习熟悉度、英文发音与语音缓存。
- 划词菜单支持跨书搜索，按书分组展示全书架命中结果，可折叠、分批展开，并支持跳转后返回最初来源书页。
- 划词菜单支持显示模式、大小、显示/隐藏和拖拽排序设置。

### 调试设置页

- 隐藏入口：关于页连续点击版本号 5 次，或主窗口搜索框输入 `--debug-ui`。
- 后台任务开关：自动导入、封面预加载、全文索引、语义索引、同步、更新检查、TTS 缓存、生词本轮询。
- 阅读页开关：阅读统计上报、已读字数检测、页数测量、沉浸模式、跨书搜索、脚注弹窗。
- 性能面板：展示最近启动耗时、封面加载耗时、索引耗时、同步耗时和最近一次卡顿任务。
- 安全模式：一键关闭后台任务，只保留书架和阅读，用于排查卡顿。
- 导出诊断：导出最近日志、调试开关状态、版本号、数据库大小和书籍数量等信息。

### 搜索与语义

- 书架全文检索：SQLite 倒排索引 + 逐章文本缓存 + 字节级匹配。
- 语义检索：`bge-small-zh-v1.5` + fastembed / ONNX，分片 HNSW 加速，支持相似段落推荐。

## 五、数据持久化

- 书库：`%APPDATA%/ebook-reader/library.json`
- 封面缓存：`%LOCALAPPDATA%/ebook-reader/covers/cover_<id>.png`
- 搜索历史：前端 `localStorage`
- 生词本：`%APPDATA%/ebook-reader/vocab.json`
- SQLite：`%APPDATA%/ebook-reader/reader.db`
  - `metadata`：设备 ID、同步服务器、账号、token、上次同步时间等本机元数据
  - `entities`：统一同步实体，字段包含 `kind / id / json / updated_at / deleted_at / device_id / sync_version`
  - `keyword_postings`：书架全文搜索倒排索引
- 轻数据包导入 / 导出：导出书籍元数据、进度、高亮、批注、书签、生词本、阅读统计和设置；不包含电子书文件本体、语音缓存、语义索引和程序本体。

## 六、构建与发布

检查：

```powershell
powershell -ExecutionPolicy Bypass -File scripts/check.ps1
```

公开 CI：

- `.github/workflows/ci.yml` 在 `main` 分支 push / pull request 时运行。
- CI 与本地开发共用 `scripts/check.ps1`，避免“本地能过、发布说明无公开流水线背书”。

发布单文件绿色版：

```powershell
powershell -ExecutionPolicy Bypass -File scripts/build-release.ps1
```

输出：

- 项目目录：`鲲鹏阅读器.exe`
- 桌面：`鲲鹏阅读器.exe`
- 中间产物：`target/release/ebook-reader-tauri.exe`

安装包：

```powershell
cargo tauri build
```

输出：`target/release/bundle/nsis/`。

发布到 GitHub Release 时，资产只上传：

- 单文件绿色版：`鲲鹏阅读器.exe`
- Windows 安装包：NSIS `.exe`

源码仓库不提交 exe，根目录本地副本由 `.gitignore` 排除。

# 电子书阅读器（Tauri 版）开发文档

> 一个面向 Windows 的高性能本地电子书阅读器。书架与阅读页相互独立、EPUB 原生渲染、按章虚拟化加载，大书秒开。

---

## 一、技术栈

| 层 | 选型 | 说明 |
|----|------|------|
| 后端 / 框架 | **Rust + Tauri 2.x** | 业务逻辑、文件解析、窗口管理跑在 Rust 端 |
| 前端 / UI | **系统 WebView2 + 原生 HTML/CSS/JS** | 不引入前端框架，纯手写，体积小、启动快 |
| 进程模型 | Tauri 多窗口（`WebviewWindowBuilder`） | 书架窗口与每个阅读窗口彼此独立，关主窗口不影响阅读窗口 |
| 数据传递 | 自定义 URI 协议 + `#[tauri::command]` | 见下文「数据通路」 |

### 主要依赖（Cargo.toml）

- `tauri` (2, 含 `devtools`) —— 主框架，F12 可开调试器
- `tauri-plugin-dialog` —— 文件选择 / 导入对话框
- `serde` / `serde_json` —— 书库持久化与前后端 JSON 交换
- `epub` (2.1.5) —— EPUB 解析（spine / toc / resources / cover / metadata）
- `image` (0.25) —— 封面缩略图提取
- `encoding_rs` + `chardetng` —— TXT/MD 编码自动识别（GBK/UTF-8 等）
- `dirs` —— 定位配置目录与缓存目录

发布优化：`opt-level=3`、`lto=true`、`codegen-units=1`。

## 二、项目结构

```
ebook-reader-tauri/
├─ Cargo.toml            # 依赖与发布配置
├─ tauri.conf.json       # Tauri 应用配置（窗口、协议、图标）
├─ build.rs              # tauri-build
├─ capabilities/
│  └─ default.json       # 权限：windows:["*"], core:default, dialog:default
├─ src/
│  ├─ main.rs            # 命令、自定义协议、阅读页注入脚本、窗口创建
│  └─ book.rs            # 图书馆模型、持久化、封面、文本解码
└─ ui/
   ├─ index.html / app.js   # 书架（封面墙、导入、搜索、排序、统计）
   └─ reader.html / reader.js  # 阅读页（工具栏、目录、搜索、书签、设置）
```

> 备注：旧的 `../ebook-reader/`（eframe/egui 版本）仅作备份保留，已停止开发。Tauri 版是当前唯一在维护的版本。

## 三、核心架构

### 1. 数据通路

前端从一个**自定义 URI 协议**异步取数据（`register_asynchronous_uri_scheme_protocol`），路由：

| 路由 | 作用 |
|------|------|
| `/cover/<id>` | 封面图片 |
| `/book/<id>` | 阅读页外壳 HTML（含注入脚本、`window.__ID__`、`window.__CH__`） |
| `/chapter/<id>/<i>` | 单章 JSON `{head, body}`，**按需逐章拉取** |
| `/txt/<id>` | TXT/MD 正文 |
| `/res/<id>/<path>` | EPUB 内部资源（图片/CSS/字体） |
| `/pdf/<id>` | PDF 原文件 |

控制类操作（打开书、读信息、存进度、搜索、书签、改简介…）走 `#[tauri::command]`。

### 2. 关键坑与定论

- **命令必须 async**：同步命令跑在 UI 主线程，重活会卡死界面；尤其 `open_book` 从同步命令里建窗口会**死锁**。所有重命令一律 `async`。
- **u64 ID 用字符串传**：书 ID 是 u64，超过 JS 安全整数 2^53，必须以字符串注入（`window.__ID__='<id>'`），否则精度丢失导致 `/chapter` 全 404。
- **按章虚拟化是性能关键**：不一次性渲染全书，只渲染当前章（CSS 多栏分页），切章时再拉下一章。这是大书从 10 秒降到约 1 秒的核心（对标「静读天下」）。

### 3. 渲染与翻页

- CSS `column` 多栏把当前章排成「书页」，左右翻页即切栏。
- 翻页留余量，避免半行字卡在窗口边缘。
- 跨章超链接重写为 `#c<N>` / `#c<N>~<片段>`，点击可跨章跳转。
- 离屏 `#measurer`（`.rr` 类）在后台逐章测量页数，得出**全书总页数**与**全书当前页**（虚拟化本身不预分页，故靠后台测量补全）。

## 四、功能清单

### 书架（index.html / app.js）
- 封面墙展示，双击封面在**独立新窗口**打开阅读页
- 一次导入多本（对话框 + 拖拽 `tauri://drag-drop`），支持 EPUB / PDF / TXT / MD
- EPUB 导入即提取封面缩略图、书名、作者、简介、字数
- 三点菜单：随机一本 / 导入 / 阅读统计 / 全选
- 漏斗面板：排序 + 布局切换
- 动画展开式搜索框（带历史记录）
- 单击选中 + 工具栏删除
- 阅读统计弹窗（累计时长、进度等）

### 阅读页（reader.html / reader.js）
- 工具栏：目录 ☰ / 上一章 ◀ / 下一章 ▶ / 搜索 🔍 / 信息 ⓘ / 书签 🔖 / 设置 ⚙ / 进度
- **目录**：打开时定位到当前章并居中，当前章前打 ✓
- **全文搜索**：列出含关键词的句子片段（高亮 + 第N章标签），点击跳转；自有搜索历史（可单条删除），输入框禁用浏览器自动填充
- **信息弹窗**：书名/作者/格式/字数/大小，**简介可编辑**（失焦自动保存）
- **书签**：定位到 章节+章内比例，稳定可回跳
- **设置**：主题（含暗色）、字号、边距等，改动触发重新分页与重新测量
- **进度**：右上角显示「第 X/Y 章 · 全书当前页/全书总页 · 进度%」
- **续读**：记录上次章节与章内比例，下次打开回到原处
- 快捷键翻页（PgUp/PgDn）
- 阅读时长统计：每 15 秒在窗口聚焦时累加

## 五、数据持久化

- 书库：`%APPDATA%/ebook-reader/library.json`（`serde_json` 美化输出）
- 封面缓存：`%LOCALAPPDATA%/ebook-reader/covers/cover_<id>.png`
- 搜索历史：前端 `localStorage`（书架键 / 阅读页 `rsearchHistory`）

`Book` 关键字段：`path / title / format / cover / author / description / progress / resume_chapter / resume_frac / word_count / bookmarks / reading_seconds`。

## 六、构建与部署

```powershell
cd ebook-reader-tauri
cargo build --release
```

- 产物：`target/release/ebook-reader-tauri.exe`
- 桌面快捷方式自动指向最新 release exe
- 调试：运行时按 **F12** 打开 WebView 调试器

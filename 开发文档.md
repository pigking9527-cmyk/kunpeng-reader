# 电子书阅读器（Tauri 版）开发文档

> 一个面向 Windows 与 macOS 的高性能本地电子书阅读器。书架与阅读页相互独立、EPUB 原生渲染、按章/虚拟小章按需加载，大书打开更快。Windows 是主发布平台；macOS 当前提供 Apple Silicon 构建，Android 如需支持则另起工程。

---

## 一、技术栈

| 层 | 选型 | 说明 |
|----|------|------|
| 后端 / 框架 | **Rust + Tauri 2.x** | 业务逻辑、文件解析、窗口管理跑在 Rust 端 |
| 前端 / UI | **系统 WebView2 + 原生 HTML/CSS/JS** | 不引入前端框架，纯手写，体积小、启动快 |
| 进程模型 | Tauri 多窗口（`WebviewWindowBuilder`） | 书架窗口与每个阅读窗口彼此独立 |
| 数据传递 | 自定义 URI 协议 + `#[tauri::command]` | 按章、按资源异步拉取数据 |

主要依赖：`tauri`、`tauri-plugin-dialog`、`serde`、`epub`、`zip`、`image`、`encoding_rs`、`chardetng`、`flate2`、`pulldown-cmark`、`mobi`、`rusqlite`、`fastembed`、`instant-distance`、`rmp-serde`、`pdf-extract`、`lopdf`、`tokio-tungstenite`。

发布优化：v1.9.2 继续使用 `opt-level=3`、`lto="thin"`、`codegen-units=16`、`incremental=true`。固定脚本 `scripts/check.ps1` 统一执行 Rust、前端、同步服务器、安全与发布基线检查；`scripts/build-release.ps1` 构建并校验绿色版；`scripts/release.ps1` 负责 Windows NSIS、tag、GitHub Release、绿色版、安装包和 SHA-256 清单。macOS Apple Silicon 包由 GitHub Actions 构建后加入同一 Release。

## 三、核心架构

### 数据通路

前端从自定义 URI 协议异步取数据：

| 路由 | 作用 |
|------|------|
| `/cover/<id>` | 封面图片 |
| `/book/<id>` | 阅读页外壳 HTML |
| `/chapter/<id>/<i>` | 单章 JSON `{head, body}`，按需逐章拉取 |
| `/txt/<id>` | TXT / Markdown / MOBI / AZW / AZW3 正文 |
| `/res/<id>/<path>` | EPUB 内部资源 |
| `/pdf/<id>` | PDF 原文件 |

控制类操作（打开书、读信息、存进度、搜索、书签、改简介、同步等）走 `#[tauri::command]`。

### 关键定论

- 重命令必须 `async`，避免 UI 主线程卡死。
- 书 ID 是 `u64`，前端必须按字符串传递，避免超过 JS 安全整数后精度丢失。
- EPUB 大书性能依赖按章/虚拟小章按需加载：普通章节按章渲染，超大 HTML 章节按标题/段落拆成虚拟小章；EPUB 元数据和处理后的章节 HTML 会按文件时间缓存。
- EPUB 元数据解析对旧书常见的“XML 注释中包含 `--`”缺陷做只读兼容处理；不修改原书文件，真正的 XML 结构错误仍会明确返回。
- 书架启动阶段要少做磁盘扫描和逐本检查；封面、自动导入、字数统计都应延后或后台执行。
- 长任务先进入 `BackgroundTaskRegistry` 再启动工作线程；状态、日志、进度和检查点原子写入 `background-tasks.json`。耐久检查点只有完整落盘后才推进内存状态，写入失败会明确返回错误；语义索引按完成一本书的边界持久化，书内未完成批次不会伪装成可续建进度。重启时运行中任务归一为暂停并沿用原 ID，不支持续建的导入任务明确失败。
- 搜索文本、过滤结果、语义向量、HNSW 图和画像缓存统一向 `MemoryGovernor` 申请 resident/transient 许可；缓存可回收，构图预算不足先落当前分片，单本超预算明确报错。
- 恢复事务 v2 的 manifest 同时记录目标文件与原文件的大小和 SHA-256；恢复前先完整预检，SQLite 通过只读现有文件语义的 `open_existing` 打开并执行完整性检查，不会因验证失败意外新建空库。发布、重开和 WAL/SHM 清理视为一个事务，崩溃后可重复执行整组回滚，未知内容保持原位并要求人工处理。
- 备份与恢复进入全局数据维护临界区，并按 `library → stats → vocab → SQLite` 的固定顺序同时持有核心数据锁；外置词典维护共用同一维护锁，避免跨文件快照混入运行时写入。
- 同步冲突顺序固定为 `updated_at → sync_version → device_id`；同步游标、统计与确认基线按“标准化服务器地址 + 账号 ID”命名空间隔离。服务端返回逐实体 accepted/conflict/rejected，客户端只原子确认确定结算的版本，拒收项保持 dirty；登录、退出、设置和同步共用切换门闩，运行中的同步不会跨账号提交。
- 单实例在 Windows 使用 `CreateMutex`，在 macOS / Linux 使用非阻塞 `flock`；锁初始化失败时安全拒绝继续启动，第二个进程将待打开图书转发给已运行实例。
- Windows 图标显示可能被 Explorer 缓存影响，发布脚本必须校验 exe 内部图标并刷新图标缓存。
- 阅读页注入按样式、布局、高亮批注和运行时拆为四个模块，由 `src/reader_page.rs` 在编译期 `concat! + include_str!` 合并，不增加运行时请求。

## 四、功能清单

### 书架

- 批量导入 EPUB / PDF / TXT / Markdown / MOBI / AZW / AZW3。
- 封面墙、横向列表、评分、阅读进度、封面显示设置。
- 搜索框带历史记录和清空按钮；可切换普通书架过滤 / 全书架正文检索。
- 漏斗面板：排序、布局、阅读状态过滤、评分过滤。
- 账号窗口：注册、登录、退出登录、本机多账号保存、立即同步。
- 阅读统计：日 / 月 / 年 / 总视图、贡献图、统计项开关、时间 / 字数柱状图切换、图表与当日书籍列表。
- 笔记汇总：按书汇总高亮、批注、查词记录，并导出 Markdown。
- 数据包导入 / 导出：备份或迁移轻数据。

### 阅读页

- 目录、搜索、信息、朗读、批注/高亮、全屏、设置、生词本。
- EPUB 整页模式使用 CSS 多栏分页，支持单页/双页；滚动模式保持连续滚动，并使用真实正文 DOM 与底部裁剪支持单击/键盘整页翻页。
- 翻页动画支持关闭、「纸张效果（Google）」或仿真翻页，动画速度可在阅读设置中调整。
- PDF 使用 PDF.js 连续滚动与文字层，支持缩放、双页和 PDF 高亮/批注。
- 书签、续读、跨章链接、脚注弹窗、主题和排版设置。
- 离线词典支持中中 / 中英 / 英中 / 英英查询；外置词典支持 TSV / CSV / JSON、StarDict 和常见未加密 MDX，按用户优先级先查外置词典再查内置词典。
- 划词翻译支持 DeepL / Google / 百度 / 腾讯翻译配置，凭据由用户本机填写保存。
- 生词本居中浮层，双列瀑布流，支持查询次数、排序、学习熟悉度、英文发音与语音缓存。
- 划词菜单支持跨书搜索，按书分组展示包含当前书在内的全书架命中结果，可折叠、分批展开，并支持跳转后返回最初来源书页。
- 划词菜单支持显示模式、大小、显示/隐藏和拖拽排序设置。

### 调试设置页

- 隐藏入口：关于页连续点击版本号 5 次，或主窗口搜索框输入 `--debug-ui`。
- 后台任务开关：自动导入、封面预加载、全文索引、语义索引、同步、更新检查、TTS 缓存、生词本轮询。
- 阅读页开关：阅读统计上报、已读字数检测、页数测量、沉浸模式、跨书搜索、脚注弹窗。
- 性能面板：展示最近启动耗时、封面加载耗时、索引耗时、同步耗时和最近一次卡顿任务。
- 安全模式：一键关闭后台任务，只保留书架和阅读，用于排查卡顿。
- 导出诊断：导出最近日志、调试开关状态、版本号、数据库大小和书籍数量等信息。

### 搜索与语义

- 书架全文检索：压缩二进制逐章索引 + Bloom 预筛选 + 有界 LRU 缓存 + 字节级精确匹配；精确扫描兜底保证结果完整。
- 语义检索模型：BGE Small 中文（512 维，默认轻量）与 BGE Large 中文（1024 维，高精度）。两者均由 FastEmbed / ONNX 在本地运行。
- 索引通路：逐书向量 → 多中心画像冷启动筛选 → 分片 HNSW 完整检索 → 未载入分片并行暴力兜底。旧 bincode 与 HNSW v2/v3/v4 可读取并迁移，新文件使用 MessagePack、尺寸和 SHA-256 校验及原子发布。
- 构建通路：按模型维度自适应 batch；每本书提交后更新容量与检查点，暂停时丢弃当前书未完成批次但保留已完成图书；加速索引按内存预算自适应分片并使用独立 Rayon 后台线程池。

## 五、数据持久化

- 书库：`%APPDATA%/ebook-reader/library.json`
- 封面缓存：`%LOCALAPPDATA%/ebook-reader/covers/cover_<id>.png`
- 搜索历史：前端 `localStorage`
- 生词本：`%APPDATA%/ebook-reader/vocab.json`
- SQLite：`%APPDATA%/ebook-reader/reader.db`
  - `metadata`：设备 ID、同步服务器、当前账号和 token 等本机元数据；同步游标与统计按服务器 + 账号作用域保存
  - `entities`：仅保留 `book_state_v2 / vocab / reading_bucket_v2` 可移植实体
  - `sync_acknowledgements`：按服务器 + 账号保存已确认的实体版本基线，账号切换后恢复各自状态
  - WAL 自动 checkpoint，journal size limit 约 64 MB；旧版 n-gram 表在 v3 迁移中清除
- 全文索引：系统缓存目录 `ebook-reader/index`，压缩索引上限 3 GB，并自动清理孤儿/旧索引。
- 恢复点：应用配置目录 `backups`，每日自动创建且最多保留 7 个，也可在设置中手动创建。
- 后台任务：`%APPDATA%/ebook-reader/background-tasks.json`，原子保存任务 ID、状态、检查点和有界日志；耐久检查点拒绝截断或静默降级，语义索引只在完整提交一本书后写入可续建边界。
- 语义模型：系统缓存目录 `ebook-reader/models`；Small 复用根目录缓存，Large 使用独立模型子目录。
- 语义索引：系统缓存目录 `ebook-reader/semantic`，逐书向量、画像、分片图及 manifest 均带版本和完整性字段；高维模型的索引体积取决于书库段落数。
- 轻数据包导入 / 导出：导入前自动创建恢复点并立即合并到运行数据；不包含电子书文件本体、语音缓存、语义索引和程序本体。

## 六、构建与发布

检查：

```powershell
powershell -ExecutionPolicy Bypass -File scripts/check.ps1
```

公开 CI：

- `.github/workflows/ci.yml` 在 `main` 分支 push / pull request 时运行。
- CI 与本地开发共用 `scripts/check.ps1`，避免“本地能过、发布说明无公开流水线背书”。

发布单文件绿色版：

```powershell
powershell -ExecutionPolicy Bypass -File scripts/build-release.ps1
```

输出：

- 项目目录：`鲲鹏阅读器.exe`
- 桌面：`鲲鹏阅读器.exe`
- 中间产物：`target/release/ebook-reader-tauri.exe`

安装包：

```powershell
cargo tauri build
```

输出：`target/release/bundle/nsis/`。

发布到 GitHub Release 时上传：

- 单文件绿色版：`Kunpeng-Reader-vX.Y.Z-portable.exe`
- Windows 安装包：`Kunpeng-Reader-vX.Y.Z-x64-setup.exe`
- macOS App 压缩包：`Kunpeng-Reader-vX.Y.Z-macOS-Apple-Silicon.app.zip`
- macOS DMG：`Kunpeng-Reader-vX.Y.Z-macOS-Apple-Silicon.dmg`
- SHA-256 清单：`SHA256SUMS.txt`

源码仓库不提交 exe，根目录本地副本由 `.gitignore` 排除。

完整发布：

```powershell
powershell -ExecutionPolicy Bypass -File scripts/release.ps1
```

`scripts/release.ps1` 默认会读取 `Cargo.toml` 版本号，检查工作区、运行 `scripts/check.ps1`、构建绿色版和 NSIS 安装包、复制 ASCII 文件名上传资产、创建/复用 `vX.Y.Z` tag、推送 `main` 与 tag，并创建或更新 GitHub Release。macOS 产物由 `macos-build.yml` 在 `macos-14` 运行器上构建；发布前需下载其中的 `.app.zip`、`.dmg` 与校验信息，再与 Windows 文件合并上传。当前 macOS 包为临时签名，未经过 Apple Developer ID 公证。常用开关：`-SkipChecks`、`-SkipBuild`、`-SkipInstaller`、`-SkipPush`、`-SkipGitHubRelease`、`-Draft`、`-AllowDirty`。
